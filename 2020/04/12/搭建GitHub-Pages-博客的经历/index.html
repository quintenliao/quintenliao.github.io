搭建 GitHub Pages 博客的经历

Posted on 2020-04-12 In Blog Views:
Symbols count in article: 1.5k Reading time ≈ 1 mins.

　　　　　　

　　本人纯属小白，计算机一窍不通（如有错误，欢迎指正）。最初的渊源要从Markdown说起。知乎上看到了markdown标记语言，想尝试用markdown来做笔记，搜到了Siriusq 的博客Markdown写作语法。然后就开始浏览Siriusq的个人博客。对此十分惊艳，也想折腾一番（消磨时间）。此番曲折，一言难尽。

　　虽然我也已经有了GitHub账号，但是由于英语水平有限，对怎么使用GitHub有点不知所措。为了打好基础，看了B站庄七上传的GitHub“傻瓜式教学”，简单易懂，新人的不二之选。博客搭建是看了B站hojun_cn（他的博客地址）的视频教学，花费了一些功夫，建好了博客。

　　本来我也是用的nexT主题。但受到siriusq博客的影响，和自己的直观感受（鼠标在右边，人习惯性靠左边一点，扯就硬扯），想把导航页面放在右侧，又对nexT不太了解，不知道nexT其实很容易就可以实现。结果舍近求远，在Hexo上找合适的主题，还是没有自己特别满意的。

　　后来突然想起来，hojun_cn曾说过可以该nexT还可以主题，试了下正合我意。跟着官网上的教程部署了一下，左改改右改改，结果发现有一串阿拉伯文字乱码。网上查了都说是编码的问题，我又把站点配置文件改为UTF-8，但还是没有作用。折腾了好久，一怒之下把整个Blog文件直接删除了（兴许骨子里带点强迫症）。因为Node.js、Sublime Text3、Git都下载好了，直接就几个命令，就完成了初步搭建，就是git clone整个仓库的速度一言难尽，一个10MB的文件花了我十多分钟。

　　然后花了一元钱在阿里云上买了一个 top 后缀的域名（除了可以改下网址，对于建博客貌似没有别的功能了，纯小白，轻喷，貌似没有人会看到😭），根据 Siriusq 和小丁的个人博客，在 source 文件夹下新建一个next.yml文件，将配置与主题分离，直接修改next.yml，保留原主题配置文件 _config.yml 源码不变，便于下次更新时修改。具体方法可以查看小丁和Siriusq的个人博客。

　　按照他们的教程添加了字数与阅读时长统计、网站运行时间、加载进度条、加载浏览进度、鼠标点击特效等诸多功能，又注册 Google Analytic 和 Google Search Console 和 百度站长，证明网站所有权，（3月11日）并提交sitemap，结果至今还没有被收录，可能是我还没有写什么东西吧！

　　GitHub拒绝百度的爬取，又根据知乎上贾小昆的教程，把博客 repo 导入 zeit 网站，生成多个 .now.sh链接。在domain区域添加自己的域名，再添加一条A记录。具体方法请移步贾小昆的知乎专栏。百度抓取诊断还是失败，出现socket读写错误或者直接拒绝访问。另：zeit网站免费账户20G/月。

　　想着zeit不强制我改成他家的DNS服务器，说不定可以试试 zeit 和 Cloudflare 两个一起用。我又把DNS服务器改成了 Cloudflare，zei t那边的域名解析也没有解除，但是目前百度抓取诊断还是失败，Google同样也没有收录。

　　还有一个问题，就是不知道为什么博客背景变成白色了。

　　本站运行时间已达300多天//😭

　　最后还有一栏about不知道怎么写，也是放在post文件夹下面吗？格式有什么要求？头大！

特别鸣谢：庄七、hojun_cn、Sirius、小丁、贾小昆
